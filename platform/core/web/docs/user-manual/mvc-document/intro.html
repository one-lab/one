One-MVC - 框架说明

MVC（model view controller）:

  • model：表示数据和业务规则。负责封装数据和对数据的处理
  • view：用户看到的并与之交互的界面。负责数据展示以及和controller的交互
  • controller：接受用户输入，并调用模型和视图完成用户需求。负责响应视图的请求，获取请求参数并调用资源，控制跳转

使用MVC的意义在于：分层，解耦，提高可重用性和可维护性

One-MVC的原则与思想：

  • 简单、高效：希望它能让使用者们轻松，悠闲的完成工作。
  • 约定大于配置：只要使用者能够遵循一些简单通用的约定，你就会发现，你可以少写很多繁琐、重复的代码而完成的同样的功能。
