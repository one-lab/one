事务性

默认情况下，仓库实例的增删改查是事务性的

对于读操作，可以将@Transactional的readOnly属性设置为true，可以提高查询性能

如果只配置@Transactional的话将使用默认的事物（readOnly=false）

如果想配置仓库接口中某个方法的事物，可以通过设置@Transactional的各种属性来实现

通常情况下大多数的查询方法都是只读的，这时可以在接口级别将事物的readOnly属性设置为true

对于需要执行更新操作的方法，可以使用@Modifying注解，这样可以覆盖之前的只读事物，从而执行一个更新操作
