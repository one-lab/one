定义查询方法

为持久层接口创建代理对象时，有三种方式指定查询语句：

  • 解析方法名
  • 使用@Query注解
  • 命名查询
定义查询策略

同时存在多种上述查询可用时，<repositories>元素提供了query-lookup-strategy属性，此属性有以下三个值，用以指定使用哪种查询方式：

  • CREATE：通过解析方法名字来创建查询，即使有符合的命名查询，或者方法通过@Query指定的查询语句，都将会被忽略
  • USE_DECLARED_QUERY：如果方法通过@Query指定了查询语句，则使用该语句实现查询；如果没有，则查找是否定义了符合条件的命名查询， 如果找到，则使用该命名查询；如果两者都没有找到，则通过解析方 法名字来创建查询
  • CREATE_IF_NOT_FOUND：如果方法通过@Query指定了查询语句，则使用该语句实现查询；如果没有，则查找是否定义了符合条件的命名查询， 如果找到，则使用该命名查询。这是query-lookup-strategy属性的默认值

分页和排序

如果想对查询结果进行分页和排序，可以传入一个Pageable实例；如果仅仅是对查询结果排序，只需传入一个Sort实例即可
